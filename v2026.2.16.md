## 1. Soporte modo `Dark` nativo

Se ha habilitado el modo `Dark` nativo de react-mui, esto afecta directamente los componentes externos

- **react-select** el componente `FormSelect` ya tiene asociado los estilos para modo dark. sin embargo si
  requiere la implementación sin el llamado al componente se tiene dos opciones.

    1. llamar a `getSelectStyles`, solo debe enviar `useTheme` y el error si es parte de un formulario

  ```tsx
  ## llamar 
  <Select<ArticuloProps>
      key={`select-${id}`}
      styles={getSelectStyles(theme, error)} 
      theme={reactSelectThemeConfig}
  />
  ```

    2. Llamar al Hook `useMuiSelectStyles`

  ```jsx
  const customStyles = useMuiSelectStyles<OptionType, IsMulti, Group>(error, size)
  <Select
      {...others}
      placeholder={'Seleccione...'}
      styles={customStyles}
      theme={reactSelectThemeConfig}
  />
  ```


- **react-date-picker** los componente `MyDatePickerField` y `MyDateRangePickerField` ya cuentan con la
  implementación modo `dark`

- **react-data-table** se han quitado las propiedades `mrtTheme` de los componentes
  `MuiTableAdvancedOptionsProps` `MuiTableNormalOptionsProps` y `MuiTableBasicOptionsProps` y pasa la
  delegación de estilos.

## 4. Componente `AppConfirmProvider`

Se añade el nuevo componente con su respectivo `context` y hook que nos permite generar cuadros de `dialog `
de confimación, bastante parecido al `material-ui-confirm`

Para su implementación se requiere modificación de archivo `App.tsx` para añadir el provider.

```tsx
// Se añaqde el provide AppConfirmProvider
<MatxTheme>
  <AppConfirmProvider
    defaultOptions={{
      title: 'Autorización',
      content: '¿Proceder con la solicitud?',
      confirmationText: 'Aceptar',
      cancellationText: 'Cancelar',
      showInput: false,
      disableCloseOnOverlay: false,
    }}
  >
    <ConfirmProvider
      defaultOptions={{
        title: 'Confirmación',
        ....
      }}
    >
      <CssBaseline />
      {content}
      <ReloadPrompt />
    </ConfirmProvider>
  </AppConfirmProvider>
</MatxTheme>
```

La forma de uso es la siguiente:

```tsx
const { requestConfirm } = useAppConfirm()
const { reason, confirmed } = await requestConfirm({
  title: 'Cancelar nota',
  description: <Typography>vamos a destruir</Typography>,
  showInput: true,
  inputRequired: true,
  // confirmButtonColor: 'cyan',
  steps: ['paso1', 'paso2'],
})
if (confirmed) {
  console.log(reason, confirmed)
}
```

### 11. Color Utils

Se han modificado las funciones `alphaByTheme, alphaByMode` de acuerdo a lo siguiente:

```typescript
// Agregado lightFactor
export const alphaByTheme = (color: string, theme: Theme, darkFactor: number = 0.2, lightFactor?: number) => {
  // 2. Generamos clave única para el caché
  const cacheKey = `alpha-theme-${color}-${theme.palette.mode}-${darkFactor}-${lightFactor ?? darkFactor}`
  if (colorCache.has(cacheKey)) {
    return colorCache.get(cacheKey)!
  }

  if (theme.palette.mode === 'light') {
    const l = alpha(color, lightFactor ?? darkFactor)
    colorCache.set(cacheKey, l)
    return l
  }
  const d = alpha(color, darkFactor + 0.01)
  colorCache.set(cacheKey, d)
  return d
}
```
