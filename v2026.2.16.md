# ISI-TEMPLATE VERSION 2026.2.16

A continuación se detallan las principales características de la versión 2026.2.16

1. Soporte modo `dark` nativo.
2. Quitar `reactPerfectScrollBar`
3. Incremento en la paleta de colores adicionales a `primary, secondary, info, warning, error, success`

___

## 1. Soporte modo `Dark` nativo

Se ha habilitado el modo `Dark` nativo de react-mui, esto afecta directamente los componentes externos

- **react-select** el componente `FormSelect` ya tiene asociado los estilos para modo dark. sin embargo si requiere la implementación sin el llamado al componente se tiene dos opciones.

  1. llamar a `getSelectStyles`, solo debe enviar `useTheme` y el error si es parte de un formulario

  ```tsx
  ## llamar 
  <Select<ArticuloProps>
      key={`select-${id}`}
      styles={getSelectStyles(theme, error)} 
      theme={reactSelectThemeConfig}
  />
  ```

  2. Llamar al Hook `useMuiSelectStyles` 

  ```jsx
  const customStyles = useMuiSelectStyles<OptionType, IsMulti, Group>(error, size)
  <Select
      {...others}
      placeholder={'Seleccione...'}
      styles={customStyles}
      theme={reactSelectThemeConfig}
  />
  ```

  

- **react-date-picker** los componente `MyDatePickerField` y `MyDateRangePickerField` ya cuentan con la implementación modo `dark`

- **react-data-table** se han quitado las propiedades `mrtTheme` de los componentes `MuiTableAdvancedOptionsProps` `MuiTableNormalOptionsProps` y `MuiTableBasicOptionsProps` y pasa la delegación de estilos.

## 2. Quitar `reactPerfectScrollBar`

Se ha quitado y eliminado la librería `reactPerfectScrollBar` en su lugar usar el componente `StyledScrollBar` como envoltorio para visualización de **Scroll** minimalista.

Se remueve de `main.tsx`, `Sidenav.tsx` `Layout1.tsx` y en `package.json`

## 3. Incremento en la paleta de colores.

Adicionalmente a los colores `primary, secondary, info, warning, success` se han añadido los siguientes colores.

![image-20260203112646553](https://pub-8c898ae616b14a58a42439f147978559.r2.dev/typora/2026/02/typ_20260203_1770132410.png)

La forma de implementación alcanza a todos los componentes nativos de react-mui.

La paleta de colores esta disponible para los siguientes componente ``

```
Button, Chip, Fab, Badge, Checkbox, Radio, Switch, Slider, CircularProgress, LinearProgress
```

Ejemplos:

```tsx
<Button variant="contained" color="blue">
  Botón blue
</Button>
<Button variant="contained" color="cyan">
  Botón cyan
</Button>
<Chip
  label="primary"
  color="yellow"
  sx={{ color: (theme) => alpha(theme.palette.yellow.contrastText, 0.8) }}
  variant="filled"
/>
<Chip
  label="success"
  color="purple"
  sx={{ color: (theme) => alpha(theme.palette.purple.contrastText, 0.8) }}
  variant="filled"
/>
<Chip label="success" color="cyan" variant="outlined" />
```

otras formas de acceso

```js
// llamada a travez de theme
const main = theme.palette.yellow.main
const light = theme.palette.yellow.light
const dark = theme.palette.yellow.dark

// Ejemplo en componente CELL, donde alpha genera un transparencia de grado 0.7
{
    muiTableBodyCellProps: {
      sx: {
        backgroundColor: (theme) => alpha(theme.palette.blue.light, 0.7),
      },
      align: 'right',
    },
}

// otro ejemplo donde backgrounCollor pueden ser blue.light, blue.dark, blue,main
muiTableBodyCellProps: {
  sx: {
    backgroundColor: 'blue.light',
  },
  align: 'right',
},

```

> NOTA: NO SE RECOMIENDA USAR PALETA DE COLORES DE REACT-MUI EJEMPLO **green[50], pink[100], ...**, dichos colores no se modifican al momento de cambiar a modo Dark o Light
